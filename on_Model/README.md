# on_Model

This directory contains model-based simulations for studying information dynamics on networks.

In contrast to `on_Equations`, the information-theoretic quantities are not evolved directly.
Instead, they are estimated from time-series data generated by explicit dynamical models.

---

## Conceptual Role

Each module in this directory follows two steps:

1. A dynamical system generates data (time series).
2. Information-theoretic quantities are estimated from the data.

These quantities include:
- Entropy (H)
- Mutual Information (MI)
- Transfer Entropy (TE)
- Reversed Transfer Entropy (rTE)

This approach connects the abstract framework to realistic systems.

---

## Key Features

### 1. Model-Based Dynamics

- A system (e.g., gene regulatory network or oscillator network) is simulated
- Time-series data is collected
- Information variables are computed using estimators

This allows:
- validation of theoretical predictions
- comparison with real-world mechanisms

---

### 2. Estimation of Information Variables

Information quantities are estimated using methods implemented in:

    Core/Estimators/

Typical considerations include:
- choice of estimator (e.g., kNN-based)
- bias and variance trade-offs
- sensitivity to sample size

## Included Examples

### 004_ABN_for_GRN

- Gene regulatory network (GRN) model
- Simulates interactions among nodes
- Produces time-series data
- Used to estimate MI, TE, and rTE

### 005_Three_Nodes_GRN

- Three-node oscillatory network
- Demonstrates cyclic information flow
- Useful for analyzing temporal patterns

---

## How to Run

From the repository root:

```bash
python -m on_Model.004_ABN_for_GRN.main
```

or

```bash
python -m on_Model.005_Three_Nodes_GRN.main
```

---

## Output

- Results are stored in model-specific directories:

    on_Model/.../Temporal_Results/

- Recorded variables include:
  - MI
  - TE
  - rTE
  - H
  - alpha parameters

---

## Plotting

Use the utility script:

```bash
python -m Utils.plot_results --dir <result_directory> --link <node1> <node2>
```

or

```bash
python -m Utils.plot_results --dir <result_directory> --node <node>
```

---

## When to Use This Module

Use `on_Model` when you want to:

- test the framework on realistic systems
- evaluate estimator performance
- compare simulated mechanisms with information dynamics
- generate data for figures and analysis

---

## Relation to on_Equations

- `on_Equations`: direct equation-based evolution
- `on_Model`: estimation from simulated data

Together, they provide complementary validation.

---

## Notes

- Results depend on estimator configuration and sample size.
- Random seeds should be fixed for reproducibility.
- Post-analysis mode may require careful indexing of saved data.

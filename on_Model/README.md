# on_Model

This directory contains model-based simulations for investigating information dynamics on networked systems.

In contrast to "on_Equations", information-theoretic quantities are not treated as dynamical variables. Instead, they are estimated from time-series data generated by explicit stochastic or deterministic models.

---

## Conceptual Role

Each module implements a two-stage procedure:

1. Dynamical evolution of a system on a network (discrete or continuous state space)
2. Estimation of information-theoretic quantities from the resulting data

The primary quantities include:

- Entropy ($H$)
- Mutual Information ($I$)
- Transfer Entropy ($T$)
- Reversed Transfer Entropy ($\tilde{T}$)

This structure provides an operational bridge between the abstract information-dynamics framework and concrete dynamical systems.

---

## Key Features

### 1. Model-Driven Data Generation

Each model specifies an explicit update rule (e.g., Boolean, probabilistic, or continuous dynamics), from which ensemble time-series data are generated.

This enables:

- empirical validation of theoretical relations
- controlled investigation of information flow mechanisms
- comparison with biologically or physically motivated systems

---

### 2. Estimation of Information Quantities

Information measures are computed using estimators implemented in:

Core/Estimators/

Typical considerations include:

- estimator choice (e.g., binning, kNN/KSG)
- bias–variance trade-off
- dependence on ensemble size and dimensionality

---

### 3. Parameterized Interactions

Models typically include tunable parameters governing:

- internal coupling (information circulation)
- external inputs or noise sources
- interaction onset (e.g., time-dependent coupling)

These parameters allow systematic exploration of competing information flows.

---

## Included Examples

### 001_Toy_Model_A

- Cyclic discrete network with an external source
- Stochastic dynamics with internal coupling and noise
- External interaction activated after a threshold time
- Useful for probing perturbations of cyclic information flow

---

### 004_ABN_for_GRN

- Gene regulatory network (GRN) model
- Figure-8 topology with Boolean-state dynamics
- Ensemble-based analysis of information flow

---

### 005_Three_Nodes_GRN

- Three-node oscillatory network
- Continuous-state dynamics
- Information estimation via KSG-type methods
- Demonstrates multiple interaction motifs

---

### 015_Boolean_Probability_Update

- Boolean-state probabilistic update model
- Competition between internal and external information inputs
- External signal activates after a specified time
- Designed to analyze trade-offs between $T_{A \to p}$ and $T_{\mathrm{Ext} \to p}$

---

## How to Run

From the repository root:
```bash
python -m on_Model.004_ABN_for_GRN.main
```
or:
```bash
python -m on_Model.005_Three_Nodes_GRN.main
```
or:
```bash
python -m on_Model.015_Boolean_Probability_Update.main
```
---

## Output

Simulation results are stored in model-specific directories:

on_Model/.../Temporal_Results/

Recorded quantities typically include:

- $H$, $I$, $T$, $\tilde{T}$
- time-resolved estimates
- auxiliary parameters (if applicable)

---

## Plotting

Use the utility script:
```bash
python -m on_Model.Utils.plot_results --dir <result_directory> --link <node1> <node2>
```
or:
```bash
python -m on_Model.Utils.plot_results --dir <result_directory> --node <node>
```
---

When to Use This Module

Use "on_Model" to:

- test the theoretical framework on explicit dynamical systems
- evaluate estimator performance under controlled conditions
- investigate mechanisms of information flow in realistic networks
- generate data for figures and empirical analysis

---

## Relation to on_Equations

- "on_Equations": direct evolution of information-theoretic variables
- "on_Model": indirect estimation from simulated dynamics

These two approaches provide complementary perspectives on information dynamics.

---

## Notes

- Results depend sensitively on estimator choice and sample size.
- Ensemble size should be sufficiently large for stable estimation.
- Fix random seeds for reproducibility when needed.
- Care is required when interpreting time-indexed quantities (e.g., $T(t_0)$ vs. $T(t_1)$).
